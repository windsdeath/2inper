`skills/internal-pr/SKILL.md`의 월간 루틴 수행.

## 절차

1. `wiki/communities/` 안의 모든 페이지 목록 확인
2. 인자가 있으면 그 community 사용. 없으면 가장 큰(노드 수 많은) community 자동 선택
3. 해당 community 페이지의 god node와 주요 노드 목록 추출
4. 각 노드를 따라가 raw에 있는 1차 자료 (미팅 노트, 디자인 결정 기록) 일부 검토
5. **공통 패턴 / 성공 요인** 3~5개를 추출
6. 케이스스터디 초안 작성:
   - 도전 과제
   - 우리의 접근
   - 결과
   - 사용된 캐릭터·디자인 자산
   - 클라이언트 인용 (있을 때만)
7. `raw/brand-assets/case-studies/YYYY-MM-<community명>.md`로 저장

## 주의

- 클라이언트 공개 동의가 frontmatter에 명시 안 된 entity는 **반드시 익명화** (회사명·로고·구체 수치 모두)
- 캐릭터 이미지 사용 가능성은 [[wiki/concepts/캐릭터 IP 가이드]] 기준으로 판정 — 의심되면 [[skills/legal-ip/SKILL]] 검토 요청 메시지를 초안 끝에 추가
- 게시·발행은 절대 자동 진행 금지. 항상 사람 검수 후 발행.
